1. 데이터 사이언스
- 정의: 데이터 마이닝과 유사하게 정형, 비정형 형태를 포함한 다양한 데이터로부터 
       지식과 인사이트를 추출하는데 과학적 방법론, 프로세스, 알고리즘, 시스템을 동원하는 융합분야

    1.1 몇 가지 오해
    - 데이터 사이언스에서 가장 중요한 것은 인공지능, 딥러닝이다
        -> 데이터 엔지니어링(데이터를 모으고 옮기고 저장하여 정리하는 것), 분석, A/B 테스트, 인공지능이 차례로 쌓여야 한다
    - 데이터 사이언티스트들은 왠지 하루종일 컴퓨터랑 수학만 하는 사람일 것 같다
        -> 가치를 더할 수 있는 문제를 데이터로 해결하는 것이 목표이기에 인사이트, 커뮤니케이션, 프로그래밍 등 모두 중요하다
    
    1.2 파이썬을 쓰는 이유
    - 데이터 사이언스에 사용하는 언어는 R과 파이썬이 대표적

        1.2.1 R
        - 통계를 위해 만들어진 언어
        - 데이터 분석의 도구가 잘 갖춰짐
        - 통계와 시각화만을 위한 툴이기에 한계
        - 연구 분야에선 파이썬을 능가하는 도구들이 많기에 많이 연구 분야에서 많이 사용

        1.2.2 Python
        - 다양한 용도로 만들어진 언어
        - 데이터 분석의 도구가 평범
        - 다른 분야로 넘어가거나 다른 분야에 데이터 사이언스를 접목하기 유리
        - Pandas, Numpy, Tensorflow 등의 라이브러리가 나오면서 2016년부터 데이터 사이언스 분야에서 R을 앞지르고 격차를 벌리고 있음

    1.3 데이터 사이언스 프로세스
        1.3.1 문제 정의하기
        - 해결하고자 하는 문제를 정의
        - 목표 설정, 기간 설정, 평가 방법 설정, 필요한 데이터 설정 등

        1.3.2 데이터 모으기
        - 필요한 데이터를 모을 수 있는 방법을 찾기
        - 누군가 이미 모아 놓은 데이터, 공공 기관 등에서 배포한 자료, 직접 데이터 수집을 할 수도 있음

        1.3.3 데이터 다듬기
        - 데이터의 퀄리트를 높여서 의미 있는 분석이 가능하게 한다
        - 데이터를 관찰하여 오류를 제거하고 정리하는 것

        1.3.4 데이터 분석하기
        - 준비된 데이터로부터 의미를 찾는다
        - 데이터를 파악하고 변형하여 통계를 내고 분석, 인사이트 발견

        1.3.5 커뮤니케이션
        - 분석 결과를 다른 사람들에게 전달
        - 다양한 시각화를 통해 다른 이들에게 원활하게 전달하고 토의해야 한다
