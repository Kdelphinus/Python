1. 개요
- CLI 환경에서 사용하는 텍스트 에디터
- 키보드로만 작동

2. 4가지 사용모드
- 일반 모드: 커서 이동, 텍스 붙여넣기, 텍스트 삭제, 작업 취소
- 입력 모드: 텍스트 입력
- 비주얼 모드: 텍스트 블록 지정, 텍스트 복사
- 명령 모드: 내용 저장, vim 종료, 특정 텍스트 검색 및 치환
- 처음엔 일반 모드: a 또는 i키를 누르면 입력 모드, v를 누르면 비주얼 모드, :키를 누르면 명령 모드
- 각 모드에서 esc를 누르면 일반 모드로 돌아감
- 다른 모드로 넘어갈 땐, 일반 모드를 항상 거쳐야 함

3. 실행
- vim: 입력하면 실행됨
- vim '파일이름': vim으로 파일 실행

4. 입력 모드
- i키를 눌러서 들어갈 때: insert의 줄임말, 입력모드로 전환
- a키를 눌러서 들어갈 때: append의 줄임말, 커서를 한 칸 뒤로 옮기고 입력 모드로 전환
- I키를 눌러서 들어갈 때: 커서가 첫번째 칸으로 돌아가면서 입력 모드로 전환
- A키를 눌러서 들어갈 때: 커서가 맨 마지막 칸으로 이동하면서 입력 모드로 전환
- o키를 눌러서 들어갈 때: 커서가 다음 줄로 넘어가면서 입력 모드로 전환
- O키를 눌러서 들어갈 때: 커서 위에 빈 줄을 만들고 그곳에 커서 생성

5. 명령 모드
    5.1 :키를 눌러서 들어감
    - w '파일이름': 파일이름으로 저장(L은 줄수, C는 문자수)
    - q: vim 종료
    - w: 이미 저장된 파일을 다시 저장할 땐, 이름을 안 적어도 됨
    - wq: 저장과 동시에 종료
    - q!:저장되지 않고 종료(!는 강제 실행을 의미)
    - s/바꾸고 싶은 텍스트/변경된 내용: 첫 번째 바꾸고 싶은 텍스트를 바꿈
    - %s/바꾸고 싶은 텍스트/변경된 내용/: 모든 줄의 첫번째 텍스트만 바꿈
    - %s/바꾸고 싶은 텍스트/변경된 내용/g: 모든 텍스트에서 변경을 진행
    - %s/바꾸고 싶은 텍스트/변경된 내용/gc: 바꿀 내용의 텍스트들을 하나씩 지나가며 바꿀 것인지 물어봄(y를 누르면 바꾸고 n을 누르면 안 바꿈)

    5.2 /키를 눌러서 들어감
    - 검색할 텍스트를 입력: 첫번째 텍스트 첫 글자로 커서가 이동
    - n: 다음 텍스트로
    - N: 이전 텍스트로

6. 일반 모드
    6.1 커서 이동
    - 방향키
    - h: 왼쪽, j: 아래, k: 위, l: 오른쪽 (예전 컴퓨터에 방향키가 없어서 생겼던 것)
    - 이동하고 싶은 칸 수 + 알파벳 키: 그 방향만큼 이동
    - ctrl + g: 커서가 있는 줄 수와 그 줄에서 위치를 보여줌
    - 0: 그 줄의 첫번째 칸으로 이동
    - $: 그 줄의 마지막 칸으로 이동
    - gg: 파일의 맨 처음줄로
    - G: 파일의 마지막 줄로

    6.2 텍스트 삭제
    - x: 텍스트를 한 칸씩 삭제
    - 삭제할 칸 수  + x: 그 수만큼 커서부터 삭제
    - dd: 커서가 있던 줄을 모두 삭제
    - 삭제하고 싶은 줄 수 + dd: 커서부터 밑의 줄 수만큼 삭제
    - u: 이전 작업 복구

7. 비주얼 모드
- v: 비주얼 모드 접근
- 커서, 혹은 알파벳으로 옮기면 드래그 됨
- x: 블록 지정된 부분이 삭제하고 일반 모드로 돌아감
- y: 복사 후, 일반모드로
- p: 일반모드에서 커서 다음 칸에 붙여넣기
- P: 일반모드에서 커서 이전 칸에 붙여넣기

- V: 비주얼 모드 접근, 줄 단위로 블록 처리
- p와 P는 커서가 있는 줄 전과 줄 다음에 붙여넣기
- d: 삭제, 일시적 저장(잘라내기), 그 후, 붙여넣기를 하면 됨
