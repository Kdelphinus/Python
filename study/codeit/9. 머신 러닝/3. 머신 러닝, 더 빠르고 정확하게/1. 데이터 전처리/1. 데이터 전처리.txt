1. 데이터 전처리
- 데이터를 그대로 사용하지 않고, 가공해서 모델을 학습시키는데 좀 더 좋은 형식으로 만들어주는 것


2. Feature Scaling
- 머신 러닝 모델에 사용할 입력 변수들의 크기를 조정해서 일정 범위 내에 떨어지도록 바꾸는 것
- 예시) 나이와 연봉을 다룰 때, 나이는 십의 자리인 반면 연봉은 천만 단위가 기본이다
        이것이 머신 러닝에 방해가 될 수 있기에 두 변수를 모두 일정 범위 안에 값으로 바꾸는 것
- 변수의 단위가 다르면 경사 하강법 할 때, 움직이는 방향과 범위가 다르지만
  Feature Scaling을 통해 범위를 일정하게 만들어서 경사 하강법을 빨리할 수 있게 도와준다

    2.1 min-max normalization
    - 최솟값과 최댓값을 이용하여 데이터의 크기를 0 ~ 1 사이로 만들어준다
    - (데이터 - 최솟값) / (최댓값 - 최솟값) -> normalization 한 데이터로 바뀜
        -normalization은 0과 1사이의 값으로 바꾼다는 의미
    - x_new: Feature Scaling 한 데이터, x_old: Feature Scaling 하기 전 데이터

    2.2 standardization(표준화)
    - 평균과 표준 편차(각 오차들을 제곱하여 평균을 낸 뒤 루트를 씌워주는 것) 개념이 들어감
    - x_new = (x_old - 평균) / 표준편차 -> 표준화 방법
    - 표준화를 한 데이터는 항상 평균은 0, 표준편차는 1이 된다
    - 이렇게 표준화한 데이터를 통계학에선 z-score라고 부름
    - 원래 평균 + 어느 데이터의 z-score * 원래 표준편차 = 원래 데이터
    

3. One-hot Encoding
- 머신 러닝에서 사용하는 데이터
  - 수치형(numerical): 나이, 몸무게, 키 등
  - 범주형(categorical): 혈액형, 성별 등
- 범주형 데이터가 들어올 때, 수치형 데이터로 바꾸어 머신러닝을 진행한다
- 그러나 1 2 3 4등으로 바꾸면 각 항목에 크고 작은 관계도 비교해서 더 방해가 된다
- 그래서 분류만큼 열을 만들어 들어가는 범주에만 1을 넣고 나머지는 0을 넣는 방식이 One-hot Encoding이다
