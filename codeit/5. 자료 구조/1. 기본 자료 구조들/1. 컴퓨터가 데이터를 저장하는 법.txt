1. 컴퓨터의 저장 방식
    1.1 스토리지
    - 데이터가 영구적으로 저장되는 곳
    - 음악, 영상 등
    - 데이터를 저장하고 받아오는데 오래 걸린다.
    - 정확히 언제 사용할지 모르겠는 것들을 저장

    1.2 메모리
    - 데이터가 임시로 저장하는 곳
    - pc가 꺼지면 사라지는 작업 파읻 등
    - 스토리지 용량보다 작다.
    - 데이터를 저장하고 받아오는데 빠르다.
    - 당장 사용해야 하는 것들을 저장

    1.3 스토리지와 메모리로 나누는 이유
    - 영화는 스토리지에 저장되어 있다.
    - 그러나 스토리지에서 실시간으로 꺼내오면 받아오는데 오래 걸려 끊긴다.
    - 그렇기에 메모리에 복사하여 실시간으로 꺼내온다.

    1.4 자료 구조
    - 자료 구조는 메모리를 중요하게 바라볼 것이다.


2. 메모리
    2.1 개요
    - 일정한 칸으로 나뉘어 있다.
    - 각 칸에 데이터를 저장할 수 있다.
    - 각 칸은 자신만의 주소가 있다.

    2.2 RAM(임의 접근 메모리)
    - 임의 접근: 저장 위치를 알면 접근할 때 항상 일정한 시간이 걸림
    - 시간 복잡도가 O(1)
    - 순차 접근(저장된 위치까지 가는데 한 단계씩 거쳐야 함)보다 효율적
    - 메모리 한 칸의 1byte가 담긴다.


3. 레퍼런스
    3.1 개요
    - 데이터에 접근할 수 있게 해주는 값
    - "주소"보다 조금 더 포괄적인 표현(자료 구조를 배울 땐 주소 = 레퍼런스로 봐도 무장함)
    - x = 95에서 x는 95를 담는 것이 아닌 95가 담긴 메모리 주소를 알려준다(파이썬에서).


4. id
    4.1 개요
    - id 함수를 이용하면 변수의 저장된 메모리 주소를 알 수 있다.
    - print(id(x))

    4.2 aliasing
    - 두 개의 다른 변수가 같은 것을 가리키면 두 변수의 메모리 주소도 동일할 것이다.
    - 이와 같이 여러 변수가 같은 메모리를 가리키는 것이 aliasing이다.
